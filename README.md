# Smoky Ice Shop (сайт)

## 1. ФРОНТЕНД

### 1.1 Главная страница (лендинг)

**Верхняя панель:**

- Лого сайта слева
- Кнопка (3 полоски) справа, которая открывает боковое меню справа

**Боковое меню (при клике):**

- Список категорий товаров
  - Одиночный клик по категории = всплывающее мини-окно с самым популярным товаром этой категории
  - Двойной клик = переход на страницу категории
- Кнопки "Вход" / "Регистрация" (coming soon — визуальные заглушки для демо, без полной логики)

**Основной контент на мэйн странице:**

- Сетка плиток с категориями товаров (данные подтягиваются из БД)
- Плавный эффект появления на плитках: лёгкое движение вверх (только для десктопа)
- Адаптивная вёрстка: отдельные стили для десктопа и мобильных (приоритет - мобилы!)

### 1.2 Страница категории

- Сетка плиток с товарами данной категории (из БД)
- Без поиска на этой версии (только навигация по категориям)
- Адаптивная вёрстка (десктоп + мобайл)

### 1.3 Страница товара

- Фото товара
- Цена
- Описание
- Блок "Другие товары" (рекомендации/похожие товары)
- Кнопка "Купить" - coming soon (визуальная заглушка, без реальной логики оплаты на этом этапе)

### 1.4 Личный кабинет (базовая версия)

- Coming soon пометка на полную авторизацию (email+пароль ИЛИ телефон+смс - оба варианта закладываем в архитектуру, реализуем позже, выберем, что заказчик больше предпочитает, и что мы сможем)

## 2. БЭКЕНД

### 2.1 База данных (PostgreSQL + Prisma)

**Таблица Product:**

- id
- name
- description
- price
- categoryId (связь с Category)
- imageUrl
- createdAt

**Таблица Category:**

- id
- name
- slug (для URL)

**Таблица Customer:**

- id
- name
- email / phone (в зависимости от способа регистрации)
- bonusBalance (баланс бонусов)
- createdAt

**Таблица Order, она надо для отображения заказов в кабинете:**

- id
- customerId
- items (список товаров с количеством)
- totalAmount
- paymentMethod (card / cash / bonus)
- status (например: "новый", "в обработке")
- createdAt

### 2.2 Бонусная система

- 1% от суммы каждой покупки начисляется на бонусный баланс клиента (поле bonusBalance в таблице Customer)
- Бонусами можно частично или полностью оплатить будущий заказ (если баланс достаточен)

### 2.3 Способы оплаты

Фиксируются в БД как поле paymentMethod у заказа:

- Наличные
- Карта
- Бонусы (доступно только если bonusBalance клиента ≥ суммы заказа)

Сама оплата картой/интеграция с платежками разными - coming soon пока

### 2.4 Авторизация

- Поддержка Email+пароль и Телефон+SMS закладывается в архитектуру, но в первой версии помечается как coming soon

### 2.5 Магазины сети

- Товар и цены одинаковые во всех точках сети (еще уточню)
